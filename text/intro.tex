\intro

Целью данной работы является исследование возможностей статических политик
управления памятью. Актуальность этой задачи можно обосновать следующими
соображениями: 

\begin{itemize}
\item Динамические политики восстановления памяти, такие как сборка мусора,
  требуют достаточно большое количество памяти для хранения внутренних данных
  сборщиков мусора. 
 
\item Поведение динамических алгоритмов восстановления памяти не предсказуемо,
  вследствие чего эти алгоритмы усложняют процесс тестирования программы.
  
\item Ручное управление памятью, при помощи расстановки инструкций восстановления
  памяти в тексте программы, провоцирует к созданию опасных ошибок в работе
  приложений.
\end{itemize}

Более конкретно, мы анализируем два возможных расширения интерфейса регионов:
замена стека регионов на стек очередей и замена стека регионов на стек
стеков. Каждый из этих подходов улучшает управляемость потребления памяти,
относительно стека регионов. Результатом исследования является предложенный
автором мини язык, необходимый для изучения свойств полученной системы.
